# 佳士售后系统 - 数据库设计文档

> 本文档详细描述系统的数据库表结构设计，包含17张核心表的字段定义、索引设计和表间关系。

---

## 一、设计规范

遵循《阿里巴巴Java开发手册》嵩山版数据库规约：

- 表名、字段名全小写 + 下划线分隔，不使用复数
- 表必备三字段：`id`（bigint unsigned 主键）、`create_time`（datetime）、`update_time`（datetime）
- 索引命名：主键 `pk_xxx`、唯一索引 `uk_xxx`、普通索引 `idx_xxx`
- 不使用外键与级联，外键概念在应用层解决
- 布尔字段用 `is_xxx`，类型 tinyint unsigned（1=是，0=否）
- 字符集 utf8mb4，排序规则 utf8mb4_general_ci
- 所有表使用 InnoDB 引擎

## 二、ER关系图

```
sys_company_type ──1:N── sys_company
sys_company_type ──N:N── sys_menu（通过 sys_type_code_menu，菜单上限）
sys_company ──1:N── sys_role
sys_company ──1:N── sys_region（仅总部）
sys_company ──N:N── sys_user（通过 sys_user_company）
sys_user ──N:N── sys_role（通过 sys_user_role）
sys_role ──N:N── sys_menu（通过 sys_role_menu）
sys_user ──N:N── sys_region（通过 sys_user_region）
sys_company ──N:N── sys_company（总部-一级通过 hq_first_contract）
sys_company ──1:N── sys_company（一级-二级通过 first_second_relation）
sys_role_template ──N:N── sys_menu（通过 sys_role_template_menu）
```
